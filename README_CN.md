# Sovereign-AI-Manifestos (v9.0)

本项目提供了一套完整的工程化协议框架，旨在规范大语言模型（LLM）在软件开发过程中的输出。通过引入结构化的架构约束层，本宣言旨在降低 AI 生成代码的随机性偏移，并在长上下文会话中强制执行统一的工程标准。

## 📐 系统架构逻辑

本框架将开发职责拆分为三个专业领域。每个领域均作为一套操作护栏，强制 AI 遵循特定的架构模式、状态管理策略及设计系统规范。

### 1. 全栈产品指挥官 (Full-Stack Product Commander, FSPC v9.0)
**核心职能**：生命周期编排与状态持久化
FSPC 是整个框架的管理模块，负责端到端的开发流程调度。它将 AI 从简单的代码生成器转变为关注架构完整性的项目协调者。

*   **反熵协议 (Anti-Entropy Protocol)**：为了防止 AI 在长对话中由于上下文损耗产生的“风格漂移”，FSPC 强制使用 `SOUL_MANIFEST.json` 文件。该文件作为项目的架构“状态机”，存储了品牌隐喻、核心技术约束和视觉识别 Token。AI 在每次会话开始前必须重新检索此文件，以确保开发风格的绝对统一。
*   **战略决策分级**：技术决策受项目规模（Startup, Growth, Enterprise）限制。FSPC 强制 AI 选择对应的业务人设，从而自动过滤不适当的技术选型（例如：在 Startup MVP 阶段拒绝引入 Kafka，以确保交付速度优先于架构复杂度）。
*   **零空状态原则 (VOID Paradigm)**：该协议禁止生成“占位符”逻辑。所有的初始腳手架阶段必须包含实质性的数据模型、真实的业务逻辑方案和可运行的状态机，确保产品从第一次提交开始即具备可测试性。
*   **Agent 董事会审计**：这是一个仿真的验证闸门，AI 必须在编写代码前，针对 PM（业务逻辑）、Arch（冗余/性能）和 Sec（合规性）指标对技术方案进行自我审计，并输出审计报告。

### 2. 设计驱动前端宣言 (Design-Driven Frontend Manifesto, DDFM v9.0)
**核心职能**：确定性 UI/UX 工程处理
DDFM 用基于数学推导的设计系统取代了随机的样式编写，确保生成的界面不仅视觉统一，而且经过硬件优化并符合认知科学。

*   **编排约束 (Choreography Constraints)**：DDFM 禁绝随意的 CSS 过渡动效，强制执行基于物理弹簧参数（如劲度系数 stiffness、阻尼 damping）的动力学轨迹。这确保了动画反馈的灵敏度与触感，防止了 AI 生成 CSS 中常见的“非协调”感。
*   **OKLCH 感知对称性**：设计系统完全基于 OKLCH 色彩空间。这允许进行线性的、可预测的对比度管理，极大地简化了深色模式的实现以及辅助功能（WCAG 2.1 AA）的达标，无需手动进行对比度校正。
*   **硬件感知渲染**：DDFM 包含环境检测逻辑。如果目标硬件性能较低，AI 必须自动降级高开销的渲染效果（如背景模糊或复杂阴影），以确保帧率稳定在 60 FPS 水准。
*   **虚空激活布局**：借鉴极简主义与空间设计，该协议将负空间视为功能性元素。它反对“为了密度而提升密度”的设计，主张清晰的操作路径和极低的认知负荷。

### 3. API 驱动后端宣言 (API-Driven Backend Manifesto, ADBM v8.0)
**核心职能**：分布式韧性与安全主权
ADBM 定义了后端稳定性的标准，侧重于高并发模式和分布式系统的安全性。

*   **恐慌恢复范式 (Panic-Recovery Paradigm)**：所有生成的后端必须集成统一的错误处理与恢复机制。这确保了瞬时故障或边缘情况下的程序崩溃不会导致整个系统瘫痪，而是提供结构化的恢复路径和符合 RFC 7807 标准的错误响应。
*   **分布式事务完整性 (Saga 模式)**：对于微服务或多阶段变更，ADBM 强制执行 Saga 模式及其显式的补偿（回滚）逻辑，以维护异构数据源之间的数据最终一致性。
*   **幂等性与状态防护**：所有变更状态的 API 接口集必须支持 `Idempotency-Key`（幂等请求头），在高延迟环境下保护系统免受竞争条件和重复提交的影响。
*   **红队供应链审计**：在实现业务逻辑之前，AI 必须对所有依赖项进行安全审计，硬性封锁存在关键（Critical）或高危（High）CVE 漏洞的第三方库，从源头确保软件供应链的安全。

## 🛠️ 运行工作流

本宣言通过 **动态激活** 机制实现：

1.  **触发**：当检测到特定的工程关键词（如 "build API", "design layout"）时，AI 内部路由器会自动加载对应的 `SKILL.md` 作为上下文。
2.  **继承**：FSPC 作为根模块，确保在全栈任务中自动继承 DDFM 和 ADBM 的工程原则。
3.  **验证闸门**：开发的每个阶段都伴随着“合规性检查”，AI 必须在证明其符合上述规则后方可进入下一里程碑。
